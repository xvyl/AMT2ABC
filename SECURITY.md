# Security Policy

## Supported Versions

We currently support the latest stable release of the AMT2ABC Compiler with security updates. Older versions may not receive fixes.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

We take security vulnerabilities seriously. Please **DO NOT** report security issues via public issues (GitHub or Gitee).

Instead, please send an email to **info@zylliondata.com**. Alternatively, use the **Report a vulnerability** button in the Security tab of the GitHub repository.

### What to include in your report

- A clear description of the vulnerability
- Steps to reproduce (if possible, a minimal test case)
- The impact of the vulnerability
- Any potential fix suggestions (optional)

### What to expect

- You will receive an acknowledgment within 48 hours
- We will investigate and keep you updated on progress
- Once a fix is ready, we will coordinate a disclosure date

## Preferred Languages

We prefer all communications to be in English or Chinese.

## Disclosure Policy

- We will issue a security advisory on GitHub
- We will credit the reporter (unless anonymity is requested)
- We will release a patch as soon as possible

Thank you for helping keep AMT2ABC secure.
